# DSA Interview Prep

Personal reference for DSA coding interviews — built progressively, kept consistent across two depths.

## Files

| File | Role | When to read |
|------|------|--------------|
| [handbook.md](handbook.md) | The living handbook — full explanations, rituals, mindset, Big O reasoning, communication phrases, recurring-mistakes log. | Re-read **fully every Sunday**. Read aloud. |
| [cheatsheet.md](cheatsheet.md) | Condensed quick reference — constraint table, templates, pre-coding ritual. | **Glance before every problem.** 30 seconds, build the reflex. |
| `patterns/` *(coming)* | One file per algorithmic pattern (`hashmap.md`, `sliding-window.md`, `two-pointers.md`). | Added as each pattern is drilled. |

## How to use this

- **Re-read the handbook fully every Sunday.** Don't skim. Read aloud — the phrases need to live in your speech, not just on the page.
- **Glance at the cheatsheet before every problem.** Especially the constraint table and the pre-coding ritual.
- **Update the "Recurring Mistakes" section yourself.** When a mistake is pointed out, add it with a concrete example. Your personal mistakes are more valuable than the prepopulated generic ones.
- **Don't treat the handbook as canon.** When you find a better phrasing, edit it. When you find an exception, note it. This is your document.
